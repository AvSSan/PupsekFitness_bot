# PupsekFitness KBJU Bot

Telegram-бот для личного учета КБЖУ: калории, белки, жиры и углеводы. Бот работает через polling, хранит данные в SQLite, ограничивает доступ по Telegram user ID и выгружает дневник в Excel.

## Возможности

- прием гибких текстовых записей о еде с описанием блюда;
- учет дневных норм КБЖУ и расчет остатка за день;
- настройки роста, веса и дневных норм;
- просмотр, редактирование и удаление сегодняшних записей;
- Excel-выгрузка за сегодня, 7 дней, 30 дней или весь период;
- whitelist-доступ через `.env`.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

В PowerShell активация окружения выглядит так:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
```

На Windows пакет `tzdata` из `requirements.txt` нужен для часового пояса `Asia/Vladivostok`.

Заполните `.env`:

```env
BOT_TOKEN=123456:replace-me
ALLOWED_USER_IDS=123456789,987654321
DATABASE_PATH=data/kbju.sqlite3
TZ=Asia/Vladivostok
```

Telegram user ID можно узнать, например, через `@userinfobot`.

## Запуск

```bash
python -m kbju_bot.main
```

## Формат записи еды

Первая непустая строка — название приема пищи. Дальше можно писать описание блюда, а среди следующих строк должны быть КБЖУ в любом порядке. Описание сохранится в базе и попадет в Excel-выгрузку.

```text
Завтрак
яичница
К 520
Б: 32
Ж 18
У 55
```

Можно прикрепить к сообщению фотографию еды и написать запись в подписи к фото. Бот сохранит только текст и КБЖУ, само изображение в базу не попадет.

Поддерживаются варианты `калории`, `ккал`, `белки`, `жиры`, `углеводы`, регистр не важен. Числа можно писать через точку или запятую.

## Настройки

Команда `/settings` отправляет шаблон. Ответьте одним сообщением:

```text
Рост 170
Вес 65
К 1800
Б 120
Ж 60
У 190
```
